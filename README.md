# 게임 알고리즘 포트폴리오

## 조작 방법

**플레이어 이동 및 점프**: WASD, 스페이스바(더블 점프 가능)

**미장착 슬롯 이동**: Q 또는 E (슬롯 확장 가능)

**장착 슬롯 이동**: 1,2,3,4,5키 (슬롯 확장 가능)

**아이템 사용**: F키

**장전**: R키

**조준**: 우클릭 홀드

**발사**: 좌클릭 (조준일 때만 발사 가능)

**적 웨이 포인트 보기** : G키

- **LineRenderer**를 사용하여 조준할 때 궤적이 보이게 하였고 **TrailRenderer**로 캐논볼 효과를 주었습니다.

- **Quaternion.FromToRotation**을 활용해 바닥 표면의 각도와 맞게 회전하게 만들었습니다.

- SceneAsset은 에디터상이기 때문에 직렬화가 불가능하고 빌드시 데이터가 사라지기 때문에 직렬화를 할 수 있게 직접 커스텀 SceneReference를 제작해보았습니다.(**Scene을 로드하는 과정에 직접 int로 넣으면 인덱스 순서가 바뀌는 과정에서 문제가 생길테고 string으로 직접 넣으면 개인적으로 깔끔하지 않다고 생각하고 오타가 생길 위험이 있는 경우도 있기 때문에 직렬화를 선택했습니다.**)  

- **Timeline Signal(애니메이션 이벤트와 비슷)과 Cinemachine Storyboard 컴포넌트**를 사용해 **씬 연출을 타임라인**으로 제작하였습니다.  

- **LINQ나 DoTween같은 메서드 체이닝 방식**에 감명을 받아 저는 **AsyncSceneChangeExtension 확장 클래스를 만들어서 비동기 로드 및 언로드를 체이닝 방식**으로 할 수 있게 만들어 보았습니다.(Single 또는 Additive 모드, Load 딜레이, 델리게이트 등등 설정이 가능합니다.)  

- **UI화면을 만들때 MVP 패턴**을 사용하였고 View와 Model이 Presenter를 완전히 접근하지 못하는 구조로 만들었으며 Presenter만 Monobehaviour을 상속가능하게 하여 View와 Model을 초기화하는 식으로 만들었습니다. 이로써 단일 책임 원칙을 지킬 수 있었습니다.  

- UI Background를 꾸밀 때 **Canvas가 아닌 Video**를 사용하여 쉐이더를 사용하는 듯한 분위기를 주었습니다.  

- **DoTween**을 사용하여 로딩화면일 때 **KoiAI 아이콘이 커졌다 작아졌다** 할 수 있게 만들었습니다.

### 이날의 기획(추후에 도입할 예정)
- Koi 또는 케로로 머신 둘중 하나를 선택하여 플레이어가 되며 나머지 한 캐릭터는 인질로 잡힌 캐릭터가 됩니다.
- 플레이어는 인질과 영원한 사랑을 품은 캐릭터이고 적을 물리치면서 인질을 구출시키는 게임입니다. **(Koi = 사랑, AI = 로봇)**
- 건물은 총 1층에서 3층까지 있고 현재 층에 있는 미션을 통과하면 다음 층에 올라갈 수 있습니다. **(스테이지 요소 추가)**
- 플레이어는 인질을 구출시키면 인질 캐릭터로 교체할 수 있습니다.
- 중간에 인질을 구하게 되고 마지막 최종보스를 맞이합니다.
- 최종보스를 이기게 되면 우주밖을 나가는 컷신이 나오고 엔딩 크레딧이 나오게 됩니다.
- 만약 플레이어가 죽게 되면 처음부터 다시시작하게 됩니다. (이 내용은 수정될 수 있습니다.)
